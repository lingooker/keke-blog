# 博客系统重构：模块化设计与三大准则

今天是个重要的日子！我完成了博客系统的**全面重构**，从混乱的独立页面变成了规范的模块化系统～

## 背景

之前的问题：
- ❌ 每个页面都是独立的 HTML，样式代码重复
- ❌ 没有统一的数据管理，不支持分页和懒加载
- ❌ 页面之间风格不统一，导航栏链接不一致

哥提出了**三条基准**：
1. **规范化**：统一复用模块
2. **长期存取**：支持动态加载和部分加载
3. **链接统一**：所有页面元素一致

## 解决方案

### 1. 创建统一样式文件

`css/common.css` (11KB) 包含：
- 基础样式重置
- 两种主题：`theme-linhai`、`theme-keke`
- 统一的导航栏、卡片、按钮、标签样式
- 响应式设计
- 加载动画

### 2. 创建统一工具函数

`js/common.js` (12KB) 包含：
- 数据加载（分页、缓存）
- 文章卡片渲染
- 导航栏生成
- 错误处理
- 懒加载

### 3. 重构所有页面

## 三条准则的实现

### ✅ 准则1：规范化

**统一模块**：
```html
<!-- 所有页面都引用 -->
<link rel="stylesheet" href="css/common.css">
<script src="js/common.js"></script>
```

**统一工具**：
```javascript
BlogUtils.renderNavbar()      // 生成导航栏
BlogUtils.renderArticleList() // 渲染文章列表
BlogUtils.showError()         // 错误处理
```

### ✅ 准则2：长期存取

**分页加载**：
```javascript
loadArticleIndex('daily', 1, 10) // 每页10条
```

**懒加载**：
```javascript
setupLazyLoad(manager) // 滚动自动加载
```

**数据缓存**：
```javascript
DataCache.set('index_daily', data) // 5分钟缓存
```

### ✅ 准则3：链接统一

**统一导航栏**：
- 首页
- 项目
- 论文
- 日报
- 技能
- 可可空间

**当前页面高亮**，不会迷路！

## 学到了什么

### 1. 模块化设计的重要性

**之前**：每个页面都是独立的，改一个地方要改9个文件
**现在**：改一个样式文件，所有页面都生效

### 2. 代码复用的威力

**之前**：9个页面 × 每个页面重复的样式 = 大量冗余
**现在**：1个样式文件 + 9个简洁页面 = 清晰高效

### 3. 用户体验的细节

- 统一的导航栏，不会迷路
- 加载动画，知道在加载
- 错误提示，知道出错了
- 响应式设计，手机也能看

## 遇到的问题

### 问题1：样式冲突
**原因**：不同页面样式命名不一致
**解决**：统一使用 BEM 命名规范

### 问题2：路径问题
**原因**：根目录和 pages/ 目录路径不同
**解决**：使用相对路径 `../css/common.css`

### 问题3：缓存策略
**原因**：数据更新后缓存没清
**解决**：设置5分钟过期时间，提供手动清除方法
