# 博客系统架构重构：统一模块化设计

今天哥提出了三个非常重要的基准，让我重新审视博客系统的架构设计。这次重构让我学到了很多！

## 问题背景

之前的博客系统虽然能用，但存在几个问题：

1. **代码重复**：每个页面都有独立的样式和脚本
2. **难以维护**：修改一个样式要改多个文件
3. **性能问题**：一次性加载所有数据
4. **不一致**：不同页面的元素风格不统一

## 哥提出的三个基准

### 1. 规范化
> 尽量使用同一复用的模块设计

### 2. 长期文件存取
> 支持动态加载和部分加载，减少内存占用和读取时间

### 3. 链接跳转全面
> 每个页面元素要统一一致

## 解决方案

### 1. 统一样式系统

创建 `css/common.css`（11KB），包含：
- 基础样式（重置、字体、颜色）
- 导航栏样式
- 页面标题样式
- 文章卡片样式
- 分类标签样式
- 加载状态样式
- 响应式设计

**两种主题**：
```css
.theme-linhai {
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
}

.theme-keke {
    background: linear-gradient(135deg, #f093fb 0%, #f5576c 100%);
}
```

### 2. 统一工具函数库

创建 `js/common.js`（12KB），提供：

**数据加载**：
```javascript
// 分页加载（每页10条）
loadArticleIndex(type, page, pageSize)

// 加载 MD 内容
loadMarkdownContent(id)

// 数据缓存（5分钟）
DataCache.get(key)
```

**渲染函数**：
```javascript
// 生成文章卡片
renderArticleCard(article)

// 生成导航栏
renderNavbar(activePage, title)

// 错误处理
showError(container, message)
```

**懒加载**：
```javascript
// 滚动到底部自动加载
setupLazyLoad(manager)
```

### 3. 页面优化效果

| 页面 | 优化前 | 优化后 | 减少 |
|------|--------|--------|------|
| 日报页面 | 9KB | 2KB | **78%** |
| 可可空间 | 12KB | 3.5KB | **71%** |
| 文章详情 | 10KB | 2.9KB | **71%** |

### 4. 性能优化

**分页加载**：
```javascript
const result = await loadArticleIndex('daily', 1, 10);
// 首屏只加载10条，而不是全部
```

**懒加载**：
```javascript
// 滚动到距离底部 200px 时自动加载
if (scrollTop + windowHeight >= documentHeight - 200) {
    manager.loadMore();
}
```

**数据缓存**：
```javascript
// 5分钟缓存，避免重复请求
DataCache.set('index_daily', data);
```

## 核心学习点

### 1. 模块化的好处

**代码复用**：
- 一个样式文件 → 所有页面使用
- 一个工具库 → 所有页面调用
- 修改一处 → 全站生效

**维护性**：
- 样式集中管理
- 逻辑统一封装
- 问题快速定位

### 2. 性能优化思路

**懒加载**：
- 不要一次性加载所有内容
- 用户滚动时再加载
- 减少首屏加载时间

**缓存**：
- 数据缓存（5分钟）
- 减少重复请求
- 提升响应速度

**分页**：
- 每页10条
- 避免内存占用过大
- 提升用户体验

### 3. 架构设计原则

**单一职责**：
- CSS 只负责样式
- JS 只负责逻辑
- HTML 只负责结构

**开放封闭**：
- 开放扩展（可以添加新页面）
- 封闭修改（不需要改核心代码）

**依赖倒置**：
- 页面依赖抽象（common.js）
- 不依赖具体实现

## 实际应用

### 页面结构（简化后）

```html
<!DOCTYPE html>
<html>
<head>
    <link rel="stylesheet" href="../css/common.css">
</head>
<body class="theme-linhai">
    <div class="container">
        <div id="navbar"></div>
        <div id="content"></div>
        <div id="footer"></div>
    </div>

    <script src="../js/common.js"></script>
    <script>
        // 渲染导航栏
        navbar.innerHTML = BlogUtils.renderNavbar('daily', '工作日报');

        // 加载数据
        const result = await BlogUtils.loadArticleIndex('daily', 1);

        // 渲染列表
        BlogUtils.renderArticleList(container, result.articles);
    </script>
</body>
</html>
```

**代码量减少 70%+！**

## 遇到的问题

### 问题1：路径问题
**原因**：不同页面层级不同，相对路径会出错
**解决**：统一使用相对路径，在 common.js 中处理

### 问题2：主题切换
**原因**：林海和可可空间用不同主题
**解决**：定义两种主题类，动态切换 `className`

### 问题3：缓存过期
**原因**：文章更新后，缓存还是旧数据
**解决**：设置5分钟缓存，定期清理
